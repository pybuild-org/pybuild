# Тег xml

```xml
<xml></xml>
```

Заполнитель без какой-либо пользы, не занимает стек, может быть записан произвольное количество раз и на произвольных уровнях.

# Тег use

```xml
<use file="" />
```

Включает внешний XML‑контент и сразу же выполняет его в текущем контексте.

`file` параметр может быть путем к файлу или ссылкой, начинающейся с `http(s)://`

Если не указать суффикс `.xml`, тег `use` автоматически добавит его

# Тег config

```xml
<config [type="group"] name=""><config>
```

Тег для объявления конфигурации, `name` указывает имя конфигурации.

`type="group"` является необязательным и используется для объявления групповой конфигурации; в этом случае тег `config` состоит из нескольких дочерних тегов `config`.

# Тег prop

```xml
<prop name="" value="" />
```

Тег для указания конкретного значения параметра конфигурации; `name` задаёт имя поля, `value` — значение поля.

# Тег run

```xml
<run job="" | command="" />
```

Тег для выполнения задачи; выбирается один из параметров `job` или `command`.

`job` указывает имя задачи, используемую для выполнения преднастроенной задачи.

`command` задаёт содержание команды, используемую для выполнения системной команды.

# Тег log

```xml
<log></log>
```

Тег для вывода одной строки журнала.
