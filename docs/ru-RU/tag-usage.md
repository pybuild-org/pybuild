# xml тег

```xml
<xml></xml>
```

Заполнитель без какой‑либо функции, не занимает стек, может быть записан произвольное количество раз и на произвольном уровне вложенности.

# use тег

```xml
<use file="" />
```

Подключает внешний XML‑контент и сразу же исполняет его в текущем контексте.

Параметр `file` может быть путём к файлу или ссылкой, начинающейся с `http(s)://`.

Если суффикс `.xml` не указан, тег `use` добавит его автоматически.

# config тег

```xml
<config [type="group"] name=""><config>
```

Тег для объявления конфигурации; атрибут `name` задаёт имя конфигурации.

`type="group"` — необязательный атрибут, указывающий, что это групповая конфигурация; в этом случае тег `config` состоит из нескольких дочерних тегов `config`.

# prop тег

```xml
<prop name="" value="" />
```

Тег для задания конкретного значения параметра конфигурации; `name` задаёт имя поля, `value` — его значение.

# run тег

```xml
<run job="" | command="" />
```

Тег для выполнения задачи; выбирается один из атрибутов `job` или `command`.

`job` указывает имя предопределённой задачи, которую следует выполнить.

`command` задаёт команду системы, которую необходимо выполнить.

# log тег

```xml
<log></log>
```

Тег для вывода одной строки в журнал.
