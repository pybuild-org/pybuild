# xml тег

```xml
<xml></xml>
```

Плейсхолдер без какой-либо пользы, не занимает стек, может быть написан произвольное количество раз, произвольное количество уровней

# define тег

```xml
<define name="" value="" />
```

Тег, используемый для определения псевдонима значения, `name` объявляет псевдоним, `value` объявляет значение

# use тег

```xml
<use src="" />
```

Включает внешний XML‑контент и сразу же выполняет его в текущем контексте

`src` параметр может быть файловым путём или ссылкой, начинающейся с `http(s)://`

`src` параметр может использовать псевдоним, определённый в теге define, через `{name}`

Если суффикс `.xml` не указан, тег `use` добавит его автоматически

# config тег

```xml
<config [type="group"] name=""><config>
```

Тег, используемый для объявления конфигурации, `name` задаёт имя конфигурации

`type="group"` является опциональным и объявляет групповую конфигурацию; в этом случае тег `config` состоит из нескольких дочерних тегов `config`

# prop тег

```xml
<prop name="" value="" />
```

Тег, используемый для указания конкретного значения конфигурационного параметра, `name` задаёт имя поля, `value` задаёт значение поля

# run тег

```xml
<run job="" | command="" />
```

Тег, используемый для выполнения задачи; выбирается один из `job` или `command`

`job` задаёт имя задачи, используемую для выполнения преднастроенной задачи

`command` задаёт содержание команды, используемую для выполнения системной команды

# log тег

```xml
<log></log>
```

Тег, используемый для вывода одной строки журнала
