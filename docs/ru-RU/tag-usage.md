# xml тег

```xml
<xml></xml>
```

Плейсхолдер без какой-либо пользы, не занимает стек, может быть записан произвольное количество раз, произвольное количество уровней

# use тег

```xml
<use file="" />
```

Включает внешний XML‑контент и сразу же выполняет его в текущем контексте

`file` параметр может быть путем к файлу или ссылкой, начинающейся с `http(s)://`

Если не указать расширение `.xml`, тег `use` автоматически добавит его

# config тег

```xml
<config [type="group"] name=""><config>
```

Тег для объявления конфигурации, `name` задаёт имя конфигурации

`type="group"` опционально, используется для объявления групповой конфигурации; в этом случае тег `config` состоит из нескольких дочерних тегов `config`

# prop тег

```xml
<prop name="" value="" />
```

Тег для указания конкретного значения параметра конфигурации, `name` задаёт имя поля, `value` — значение поля

# run тег

```xml
<run job="" | command="" />
```

Тег для выполнения задачи, выбирается один из `job` или `command`

`job` задаёт имя задачи, используется для выполнения предварительно настроенной задачи

`command` задаёт содержание команды, используется для выполнения системной команды

# log тег

```xml
<log></log>
```

Тег для вывода одной строки журнала
