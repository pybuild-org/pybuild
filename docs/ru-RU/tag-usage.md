# xml 标签

```xml
<xml></xml>
```

Заполнитель без какой‑либо функции, не занимает стек, может быть записан произвольное количество раз и на произвольных уровнях вложенности.

# define 标签

```xml
<define name="" value="" />
```

Тег для определения псевдонима значения: `name` объявляет псевдоним, `value` объявляет значение.

Параметр `value` можно использовать в предыдущих тегах `define` через `{name}`.

# use 标签

```xml
<use src="" />
```

Подключает внешний XML‑контент и сразу же выполняет его в текущем контексте.

Параметр `src` может быть путём к файлу или ссылкой, начинающейся с `http(s)://`.

Параметр `src` можно задавать через `{name}`, используя псевдонимы, объявленные в теге `define`.

Если суффикс `.xml` не указан, тег `use` добавит его автоматически.

# config 标签

```xml
<config [type="group"] name=""><config>
```

Тег для объявления конфигурации; `name` задаёт название конфигурации.

`type="group"` — необязательный атрибут, указывающий, что это групповая конфигурация, состоящая из нескольких вложенных тегов `config`.

[Общие параметры конфигурации](./common-config.md)

# prop 标签

```xml
<prop name="" value="" />
```

Тег для указания конкретного значения параметра конфигурации: `name` задаёт название поля, `value` — его значение.

# run 标签

```xml
<run job="" | command="" />
```

Тег для выполнения задачи; выбирается один из атрибутов: `job` или `command`.

`job` объявляет название преднастроенной задачи, которую следует выполнить.

`command` задаёт команду системы, которую следует выполнить.

# log 标签

```xml
<log></log>
```

Тег для вывода одной строки в журнал.
