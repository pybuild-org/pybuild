# xml タグ

```xml
<xml></xml>
```

何の用途もないプレースホルダーで、スタックを使用せず、任意の回数、任意の階層で記述できます。

# use タグ

```xml
<use file="" />
```

外部の XML コンテンツを取り込み、現在のコンテキストで即座に実行します。

`file` パラメータはファイルパス、または `http(s)://` で始まるリンクのいずれかにできます。

`.xml` 拡張子が付いていない場合、`use` タグは自動的に付加します。

# config タグ

```xml
<config [type="group"] name=""><config>
```

設定を宣言するためのタグで、`name` は設定名を宣言します。

`type="group"` はオプションで、グループ設定を宣言するために使用します。この場合、`config` タグは複数の子 `config` タグによって構成されます。

# prop タグ

```xml
<prop name="" value="" />
```

設定項目の具体的な値を宣言するタグで、`name` はフィールド名、`value` はフィールド値を宣言します。

# run タグ

```xml
<run job="" | command="" />
```

タスクを実行するためのタグで、`job` と `command` のいずれか一方を指定します。

`job` はタスク名を宣言し、事前に設定されたタスクを実行します。

`command` はコマンド内容を宣言し、システムコマンドを実行します。

# log タグ

```xml
<log></log>
```

1 行のログを出力します。
