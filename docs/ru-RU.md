#Использование

## Локальный запуск

Скачайте предварительно скомпилированный бинарный файл с https://github.com/pybuild-org/pybuild/releases

Запустите `pybuild`, чтобы начать компиляцию `target.xml`

Укажите файл конфигурации `pybuild custom.xml`

## Запуск через GitHub Action

```yaml
- name: setup pybuild
  uses: pybuild-org/pybuild@main
  with: # optional
      version: 'latest' # default
      goos: 'linux' # default
      goarch: 'amd64' # default
```

Затем команда `pybuild` будет доступна в последующих задачах

# Использование тегов

## Тег xml

```xml
<xml></xml>
```

Бесполезный placeholder, не занимает стек, можно писать любое количество раз, любые уровни вложенности

## Тег use

```xml
<use file="" />
```

Вставляет внешний XML‑контент и сразу выполняет его в текущем контексте

Параметр `file` может быть путём к файлу или ссылкой, начинающейся с http(s)://

Если не указано расширение `.xml`, тег `use` автоматически добавит его

## Тег config

```xml
<config [type="group"] name=""><config>
```

Тег для объявления конфигурации, атрибут `name` задаёт имя конфигурации

Атрибут `type="group"` является необязательным, используется для объявления групповой конфигурации; в этом случае тег `config` объявляется совместно с несколькими дочерними тегами `config`

## Тег prop

```xml
<prop name="" value="" />
```

Тег для объявления конкретного значения конфигурационного параметра, где `name` — имя поля, `value` — значение поля

## Тег run

```xml
<run job="" | command="" />
```

Тег для выполнения задачи; атрибуты `job` и `command` являются взаимоисключающими

`job` указывает имя задачи, которая будет выполнена как предварительно настроенная задача

`command` указывает содержимое команды, которая будет выполнена как системная команда

## Тег log

```xml
<log></log>
```

Тег для вывода одной строки лога
