#Использование

## Локальный запуск

Скачайте предварительно скомпилированные бинарные файлы из https://github.com/pybuild-org/pybuild/releases

Запустите `pybuild` для начала компиляции `target.xml`

Укажите файл конфигурации `pybuild custom.xml`

## Запуск в GitHub Actions

```yaml
- name: setup pybuild
  uses: pybuild-org/pybuild@main
  with: # optional
      version: 'latest' # default
      goos: 'linux' # default
      goarch: 'amd64' # default
```

Затем команда `pybuild` становится доступной в последующих задачах

# Использование тегов

## Тег xml

```xml
<xml></xml>
```

Это бесполезный заполнитель, не занимает стек, его можно писать произвольное количество раз и на любой глубине вложенности

## Тег use

```xml
<use file="" />
```

Вставляет внешний фрагмент XML и сразу выполняет его в текущем контексте

Параметр `file` может быть путём к файлу или ссылкой, начинающейся с http(s)://

Если не указано расширение `.xml`, тег `use` автоматически добавит его

## Тег config

```xml
<config [type="group"] name=""><config>
```

Используется для объявления конфигурации, атрибут `name` задаёт имя конфигурации

`type="group"` является необязательным и используется для объявления групповой конфигурации; в этом случае данный тег `config` объявляется несколькими дочерними тегами `config`

## Тег prop

```xml
<prop name="" value="" />
```

Используется для объявления конкретного значения свойства конфигурации; атрибут `name` задаёт имя поля, а `value` — значение поля

## Тег run

```xml
<run job="" | command="" />
```

Используется для выполнения задачи; атрибуты `job` и `command` являются взаимоисключающими

`job` задаёт имя задачи для выполнения предварительно настроенной задачи

`command` задаёт содержание команды для выполнения системной команды

## Тег log

```xml
<log></log>
```

Используется для вывода одной строки лога
