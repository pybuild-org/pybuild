#Использование

## Локальный запуск

Скачайте предварительно скомпилированный бинарный файл с https://github.com/pybuild-org/pybuild/releases

Запустите `pybuild`, чтобы начать компиляцию `target.xml`

Укажите файл конфигурации `pybuild custom.xml`

## Запуск через GitHub Action

```yaml
- name: setup pybuild
  uses: pybuild-org/pybuild@main
  with: # optional
      version: 'latest' # default
      goos: 'linux' # default
      goarch: 'amd64' # default
```

Затем команда `pybuild` будет доступна в последующих задачах

# Использование тегов

## Тег xml

```xml
<xml></xml>
```

Бесполезный placeholder, не занимает стек, можно писать сколько угодно раз, в любой вложенности

## Тег use

```xml
<use file="" />
```

Вставляет внешний XML‑контент и сразу выполняет его в текущем контексте

Параметр `file` может быть путём к файлу или ссылкой, начинающейся с `http(s)://`

Если не указать расширение `.xml`, тег `use` автоматически добавит его

## Тег config

```xml
<config [type="group"] name=""><config>
```

Тег для объявления конфигурации, атрибут `name` задаёт имя конфигурации

Атрибут `type="group"` является необязательным и объявляет групповую конфигурацию; в этом случае тег `config` состоит из нескольких дочерних тегов `config`, которые совместно объявляют её

## Тег prop

```xml
<prop name="" value="" />
```

Тег для объявления конкретного значения конфигурационного параметра, где `name` — имя поля, `value` — его значение

## Тег run

```xml
<run job="" | command="" />
```

Тег для выполнения задачи; необходимо указать либо `job`, либо `command`

Атрибут `job` задаёт имя задачи, которая будет выполнена как предварительно настроенная задача

Атрибут `command` задаёт содержимое команды, которая будет выполнена как системная команда

## Тег log

```xml
<log></log>
```

Тег для вывода одной строки лога
